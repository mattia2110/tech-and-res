# Amendments

Amendments are sub-components of Laws that provide additional modifiers, effects, and flavor without requiring a completely new law type. They allow for fine-tuning legal systems (e.g., adding specific rights or restrictions) and can be negotiated during law enactment or political events.

## File Location
Amendments are defined in:
`common/amendments/*.txt`

## Structure

```pdx
amendment_key = {
	# Optional reference to a law that dictates default IG/Movement stances
	parent = law_key 

	# List of laws where this amendment can be attached
	allowed_laws = { 
		law_key_1
		law_key_2
	}

	# General modifiers applied to the country
	modifier = { 
		country_authority_add = 50
	}

    # Taxation-specific modifiers (added to the law's tax modifiers)
    tax_modifier_very_low = { ... }
	tax_modifier_low = { ... }
	tax_modifier_medium = { ... }
	tax_modifier_high = { ... }
	tax_modifier_very_high = { ... }

	# Connection to an Institution
	institution = institution_key
	institution_modifier = { 
		# Only applies if the country has the specified institution
		institution_investment_cost_mult = -0.1
	}

	# Trigger to check if the amendment is available for selection
	possible = { 
		always = yes
	}
	
	# Trigger (ROOT = IG) to check if an IG would sponsor this during negotiations
	# scope:approval is the IG's approval for the parent law
	would_sponsor = {
		ROOT = {
			OR = {
				is_interest_group_type = ig_rural_folk
				AND = {
					exists = scope:approval
					scope:approval > 0
				}
			}
		}
	}										

	# Multiplier for political activism generated by this amendment
	amendment_activism_multiplier = 1.0 

	# Trigger checking if the country/AI can repeal this amendment
	can_repeal = {
		legitimacy >= legitimacy_to_repeal_amendment
	}

	# Trigger for AI deciding to revoke it
	ai_will_revoke = {
		always = no
	}

	# Effects when the amendment becomes active/inactive
	on_activate = { ... }
	on_deactivate = { ... }
}
```

## Localization

Amendments use standard localization keys:
- `amendment_<key>`: The name of the amendment.
- `amendment_<key>_desc`: The flavor description.

Example:
```yaml
l_english:
 amendment_free_speech: "Freedom of Speech"
 amendment_free_speech_desc: "Ensures the right of citizens to speak freely."
```

## Template Example: "Regional Autonomy"

```pdx
amendment_regional_autonomy = {
	parent = law_governance_principles
	
	allowed_laws = {
		law_presidential_republic
		law_parliamentary_republic
		law_monarchy
	}

	modifier = {
		state_turmoil_mult = -0.1
		country_bureaucracy_add = -100
	}

	possible = {
		has_technology = tech_human_rights
	}

	would_sponsor = {
		ROOT = {
			OR = {
				is_interest_group_type = ig_intelligentsia
				is_interest_group_type = ig_rural_folk
			}
		}
	}

	can_repeal = {
		legitimacy >= 50
	}

	on_activate = {
		custom_tooltip = "Citizens feel more empowered."
	}
}
```

## Key Mechanisms

1. **Law Negotiation**: Amendments can be "proposed" by IGs during the enactment of a law if the IG `would_sponsor` it.
2. **Tax Interaction**: If an amendment is attached to a law with tax levels (like `law_per_capita_taxation`), the corresponding `tax_modifier_<level>` block will be active.
3. **Repeal Mechanics**: Unlike laws, which must be replaced by another law, amendments can be removed directly if the `can_repeal` trigger is met.
